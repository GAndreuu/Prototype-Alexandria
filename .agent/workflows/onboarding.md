# Workflow: Onboarding Completo (Total Recall)

Este workflow carrega TODO o conhecimento documentado do projeto para memória.

## Passo 1: Mapear Documentação
// turbo
1. Utilize a ferramenta `find_by_name` com pattern `**/*.md` na pasta `docs/` para encontrar todas as documentações.
2. Identifique também os arquivos `.md` na raiz (`README.md`, `STRUCTURE.md`, `task.md`, etc).

## Passo 2: Ingestão de Conhecimento
// turbo
1. Leia o conteúdo de **TODOS** os arquivos Markdown encontrados.
2. Não ignore nenhum arquivo, independentemente da profundidade.

## Passo 3: Resumo Situacional Profundo
Gere um diagnóstico completo do sistema com base em toda a documentação lida:
1. **Visão Holística**: O que é o sistema e como os módulos se conectam?
2. **Estado da Arte**: Qual o status atual de desenvolvimento de cada módulo core?
3. **Débitos Técnicos**: O que a documentação diz que falta fazer?
4. **Orientações de Trabalho**: Quais são as regras e padrões vigentes?

---

> **Aviso**: Este workflow carrega muito contexto. Use para obter "consciência plena" do projeto.
